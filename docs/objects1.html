2  Objects - Vectors, Matrices, and Arrays

In Base R, every element of data is treated as an object, a core principle introduced in Chapter 1 that underpins effective data analysis and programming. This chapter explores three fundamental R objects—vectors, matrices, and arrays—which are essential for organizing and manipulating numerical and character data. By learning to create these objects, understand their unique attributes, and perform basic operations such as subsetting and reshaping, you will build the skills to handle data with precision and flexibility. Building on Chapter 1’s introduction to R’s syntax and basic commands, this chapter provides the foundation for working with structured data, preparing you for more complex objects like data frames and lists in Chapter 3.

2.1 Vectors

Vectors are the simplest and most fundamental objects in R, serving as the building blocks for data analysis. They are one-dimensional, homogeneous collections of elements, meaning all elements must share the same data type. This section covers how to create vectors, understand their types and attributes, and perform basic operations like accessing elements and concatenation.

2.1.1 Types of Vectors

Vectors in R are atomic, meaning they cannot contain mixed data types, and come in four common types: integer, double, logical, and character. The c() function, which stands for “concatenate,” is the primary method for creating vectors. To explore a vector’s properties, use typeof() to check its storage mode (data type) and class() to confirm its structure, which for vectors is typically numeric, logical, or character. The length() function returns the number of elements in a vector, reflecting its linear structure.

v <- c(2.9, 3.5, 4.5, NA, 3, 2.4)
v
[1] 2.9 3.5 4.5  NA 3.0 2.4
typeof(v)
[1] "double"
class(v)
[1] "numeric"
length(v)
[1] 6

By default, R stores numbers as double (double-precision real numbers). To create an integer vector, append the L suffix to numbers, explicitly indicating integer values. This is often encountered in vectors generated by R functions, such as sequences. For example, functions like seq() or operations in Bioconductor packages, such as those for genomic data analysis, may produce integer vectors to optimize storage efficiency. This is particularly common when handling large datasets where integer types reduce memory usage.

i <- c(1L, 3L, 10L)
i
[1]  1  3 10
typeof(i)
[1] "integer"
class(i)
[1] "integer"

Character vectors are common when accessing text data, such as columns in a data frame read from external files. Enclose character values in quotation marks (" " or ' '), except for NA, which represents missing values.

colors <- c("red", "green", "blue", "yellow", NA, "purple")
colors
[1] "red"    "green"  "blue"   "yellow" NA       "purple"
typeof(colors)
[1] "character"
class(colors)
[1] "character"

Logical vectors, consisting of TRUE or FALSE (also T or F), often result from comparisons (covered later in this chapter). They are useful for filtering or conditional operations.

logical_vec <- c(TRUE, NA, T, F)
typeof(logical_vec)
[1] "logical"
class(logical_vec)
[1] "logical"

2.1.2 Accessing Elements in a Vector

A common task is accessing or extracting elements from a vector. This section introduces positional indexing, one of five methods for accessing elements (additional methods are covered in Chapter 4). Since vectors are linear, elements are indexed by their position (1, 2, …, N). Use square brackets [] with an index vector to select elements. An index vector specifies the positions of elements to extract from the target vector.

v <- c(2.9, 3.5, 4.5, NA, 3, 2.4) 
v[1]
[1] 2.9

To select multiple elements, create an index vector using c(). For example, to extract the first, second, and fourth elements:

v[c(1,2,4)]
[1] 2.9 3.5  NA

For consecutive elements, use the colon operator (:) to generate a sequence of positions. The following selects the second through fifth elements:

v[2:5]
[1] 3.5 4.5  NA 3.0

2.2 Attributes

Attributes are metadata associated with R objects, such as vectors, providing additional information like names or custom descriptors. They enhance data organization and accessibility, especially in data analysis tasks.

2.2.1 The Names Attribute

Vectors can have a names attribute to label elements, making them easier to reference. Use the names() function to assign or retrieve names.

v <- c(2, 3, 3.5, 6.7) 
names(v) <- c("a", "b", "c", "d")
v
  a   b   c   d 
2.0 3.0 3.5 6.7 
names(v)
[1] "a" "b" "c" "d"

Elements can be accessed using names within square brackets, offering a descriptive alternative to positional indexing.

v["a"]
a 
2 
v[c("a", "c")]
  a   c 
2.0 3.5

Names can also be assigned during vector creation using the c() function with named arguments.

v <- c(first=1, second=pi, third = sqrt(2))
v
   first   second    third 
1.000000 3.141593 1.414214

A vector with a names attribute is called a named vector, commonly used for clarity in data analysis.

2.2.2 User-defined Attributes

Beyond the names attribute, you can create custom attributes using the attr() function. This allows attaching metadata, such as creation dates or descriptions, to objects. Use attr() to set or retrieve specific attributes, and attributes() to view all attributes.

attr(v, "when") <- "Created on 6/24/25"
v
   first   second    third 
1.000000 3.141593 1.414214 
attr(,"when")
[1] "Created on 6/24/25"
attr(v, "when")
[1] "Created on 6/24/25"
attr(v, "names")
[1] "first"  "second" "third" 
attributes(v)
$names
[1] "first"  "second" "third" 

$when
[1] "Created on 6/24/25"

The attributes() function returns a list, highlighting that attributes are stored as a list object. As emphasized in Chapter 1, “everything in R is an object,” so always check the type and structure of function outputs using typeof() and class() to build proficiency.

2.3 Concatenation

Concatenation involves combining vectors of the same data type to create a larger vector. The c() function is the primary tool for this operation, essential for data manipulation tasks.

2.3.1 Concatenating Vectors

To combine vectors, use c() to append elements or vectors.

v
   first   second    third 
1.000000 3.141593 1.414214 
attr(,"when")
[1] "Created on 6/24/25"
v1 <- c(v, 3)
v1
   first   second    third          
1.000000 3.141593 1.414214 3.000000 
v2 <- c(v, v1)
v2
   first   second    third    first   second    third          
1.000000 3.141593 1.414214 1.000000 3.141593 1.414214 3.000000 
v3 <-  c(v, extra = 3)
v3
   first   second    third    extra 
1.000000 3.141593 1.414214 3.000000

The extra name in the last example assigns a label to the new element, preserving the original names from v while adding a descriptive identifier for clarity.

2.3.2 Coercion

When combining vectors of different data types, R applies coercion to convert them to a common type, following the hierarchy: logical → integer → double → character. This ensures operations proceed without errors but may affect data integrity.

For implicit coercion, R automatically converts types based on the hierarchy.

typeof(c(FALSE, 1L))
[1] "integer"
typeof(c(1L, pi))
[1] "double"
typeof(c(pi, "pi"))
[1] "character"

Explicit coercion uses as.xxx() functions (e.g., as.numeric(), as.character()) to control type conversion.

as.logical(c(FALSE, 1L))
[1] FALSE  TRUE

Coercion can lead to data loss or missing values. For example, converting doubles to integers truncates decimals, and non-numeric characters to numbers produce NA.

as.integer(c(1L, pi))
[1] 1 3
as.double(c(pi, "pi"))
Warning: NAs introduced by coercion
[1] 3.141593       NA

Be cautious with coercion to avoid unintended data loss, and always verify the resulting data type using typeof() or class().

2.4 Vectorized Operations

Vectorized operations are a cornerstone of R, enabling efficient and concise data manipulation by applying computations to entire vectors in a single expression. Unlike traditional programming languages that require explicit loops, R’s vectorized approach leverages optimized C and Fortran code, making it ideal for statistical computing and large datasets. This section explores how vectorized operations work on vectors, including element-wise arithmetic, scalar operations, and recycling, laying the groundwork for comparisons that produce logical vectors later in this chapter.

2.4.1 Element-Wise Arithmetic

When applying an arithmetic operation (e.g., +, -, *, /) to two vectors of the same length, R performs the operation element-wise, producing a new vector of the same length. Each element in the result corresponds to the operation applied to the paired elements of the input vectors.

v1 <- c(1, 2, 3)
v2 <- c(4, 5, 6)
v1 + v2
[1] 5 7 9
v1 * v2
[1]  4 10 18

In these examples, R adds or multiplies corresponding elements: \(1 + 4 =5\), \(2 + 5 = 7\), \(3 + 6 = 9\) for addition, and \(1 \times 4 = 4\), \(2 \times 5 = 10\), \(3 \times 6 = 18\) for multiplication.

2.4.2 Scalar Operations

Vectorized operations allow a single value (scalar) to be applied to every element of a vector. R automatically recycles the scalar to match the vector’s length, simplifying tasks like scaling or offsetting data values.

v1 <- c(1, 2, 3)
v1 + 5
[1] 6 7 8
v1 * 2
[1] 2 4 6

Here, R adds 5 or multiplies by 2 for each element of v1, equivalent to operating with a vector c(5, 5, 5) or c(2, 2, 2).

2.4.3 Recycling with Unequal Lengths

When vectors have different lengths, R recycles the shorter vector to match the longer one’s length. If the longer vector’s length is a multiple of the shorter one, recycling is seamless. Otherwise, R issues a warning but completes the operation.

v3 <- c(7, 8)
v1 + v3
Warning in v1 + v3: longer object length is not a multiple of shorter object
length
[1]  8 10 10

In this example, v3 is recycled as c(7, 8, 7) to match v1’s length, adding element-wise: \(1+7=8\), \(2+8=10\), \(3+7=10\). Be cautious with recycling, as mismatched lengths may lead to unintended results. Use length() to verify vector lengths before operations to avoid such issues.

2.4.4 Vectorized Functions

Many Base R functions, such as sqrt() and log(), are vectorized, applying computations to each element of a vector without loops. These are useful for transforming data in statistical analyses.

v1 <- c(1, 4, 9)
sqrt(v1)
[1] 1 2 3
log(v1)
[1] 0.000000 1.386294 2.197225

2.5 Numeric Vectors

Numeric vectors, encompassing both integer and double types, are essential for quantitative data analysis in R. While the c() function is ideal for creating small vectors without patterns, R provides specialized functions like seq(), rep(), and random number generators to efficiently create numeric vectors with specific patterns or distributions. This section explores these functions, enabling you to generate sequences, replicate values, and simulate data, building on the vector concepts from earlier sections.

2.5.1 The seq() Function

The colon operator (:) generates a sequence of integers with a step size of 1, in ascending or descending order.

2:6
[1] 2 3 4 5 6
4:-3
[1]  4  3  2  1  0 -1 -2 -3

The more frequently used method for generating a sequence of numbers is by employing the seq() function. The first two arguments, named from= and to=, define the starting and ending points of the sequence.

seq(5, 10)
[1]  5  6  7  8  9 10

The by= argument specifies the step size, while length.out= sets the desired number of elements. These arguments are mutually exclusive, as specifying both would create ambiguity in the sequence’s structure.

seq(2, 20, by=3)
[1]  2  5  8 11 14 17 20
seq(2, 20, length.out=7)
[1]  2  5  8 11 14 17 20

The seq_along() function generates a sequence of integers from 1 to the length of a given vector, useful for creating indices for data analysis tasks.

my_vector <- c("apple", "banana", "cherry")
seq_along(my_vector)
[1] 1 2 3

This sequence corresponds to the positions of elements in my_vector, facilitating tasks like iterating over vector elements.

2.5.2 The rep() Function

The rep() function replicates a vector or its elements, offering flexibility for creating repeated patterns. Its main arguments are x= (the vector to replicate) and times= (the number of repetitions).

When times is a single integer, the entire vector x is repeated that number of times.

i <- rep(2, 4)
i
[1] 2 2 2 2
x <- 1:4
y <- rep(x, 2)
y
[1] 1 2 3 4 1 2 3 4

When times is a vector of the same length as x, each element x[i] is repeated times[i] times, allowing customized repetition patterns.

z <- rep(x, i)
z
[1] 1 1 2 2 3 3 4 4
w <- rep(x, x)
w
 [1] 1 2 2 3 3 3 4 4 4 4

Use length() to verify the resulting vector’s length, especially for complex repetition patterns.

2.5.3 The Random Number Generator Functions

Random number generators in R create vectors based on probability distributions, such as uniform or normal, and are named with the prefix r (e.g., runif(), rnorm()). These functions are valuable for simulations in statistics and data analysis. Below is a table of common random number generators and their distributions.

Function Distribution
runif(n, min=0, max=1) Uniform
rnorm(n, mean=0, sd=1) Normal
rbinom(n, size, prob) Binomial
rpois(n, lambda) Poisson
rexp(n, rate = 1) Exponential
rt(n, df) T
rf(n, df1, df2) F
rchisq(n, df) Chi square

To generate 100 numbers from a normal distribution with mean 2 and standard deviation 3:

n <- rnorm(100, 2, 3)
head(n)
[1]  1.928964  1.288082  5.086042  5.236424  2.931389 -1.694067
mean(n)
[1] 1.722565
sd(n)
[1] 2.824913

The set.seed() function ensures reproducibility by fixing the random number generator’s starting point. Setting the same seed produces identical random sequences, which is crucial for debugging and sharing results.

set.seed(123)
random_vector1 <- rnorm(5, mean = 0, sd = 1)
random_vector1
[1] -0.56047565 -0.23017749  1.55870831  0.07050839  0.12928774
set.seed(123)
random_vector2 <- rnorm(5, mean = 0, sd = 1)
random_vector2
[1] -0.56047565 -0.23017749  1.55870831  0.07050839  0.12928774

Always set a seed before generating random numbers to ensure consistent results across sessions.

2.6 Logical Vectors

Logical vectors, consisting of TRUE and FALSE values, are fundamental for conditional operations and data filtering in R. Logical Vectors are often generated through comparisons and manipulated using logical operators. This section explores how to create logical vectors with relational operators, handle missing values, apply Boolean algebra, and use logical vectors in practical data analysis tasks, such as counting or summarizing conditions.

2.6.1 Generating Logical Vectors

Logical vectors are commonly created using relational operators: <, <=, >, >=, == (exact equality), and != (exact inequality). These operators compare vectors element-wise, producing a logical vector where TRUE indicates the condition is met and FALSE indicates it is not.

v <- c(seq(2, 4), NA)
v > 3
[1] FALSE FALSE  TRUE    NA

In this example, v > 3 checks if each element exceeds 3, returning NA for missing values. Be cautious when using == with numeric vectors due to floating-point precision issues.

x <- c(1 / 49 * 49, sqrt(2)^2)
x
[1] 1 2
x == c(1, 2)
[1] FALSE FALSE

The which() function converts a logical vector to an integer vector of indices where the condition is TRUE, ignoring NA values.

which(v > 3)
[1] 3

2.6.2 Handling Missing Values

Operations involving missing values (NA) in logical vectors typically result in NA, as the outcome depends on an unknown value.

NA > 5
[1] NA
10 == NA
[1] NA
NA == NA
[1] NA

Use is.na(), introduced in Chapter 1, to identify missing values in any vector type, returning TRUE for NA and FALSE otherwise.

is.na(c(1, NA, 3))
[1] FALSE  TRUE FALSE
is.na(c("a", NA, "b"))
[1] FALSE  TRUE FALSE
is.na(c(TRUE, NA, FALSE))
[1] FALSE  TRUE FALSE

2.6.3 Logical Operators and Boolean Algebra

Logical operators &, |, and ! perform element-wise Boolean algebra on logical vectors. The & operator returns TRUE for elements where both vectors are TRUE (logical AND), | returns TRUE if at least one is TRUE (logical OR), and ! negates the vector.

h <- c(1, 10, 3.5, 8, 8)
h1 <- h > 5
h1
[1] FALSE  TRUE FALSE  TRUE  TRUE
h2 <- h == 8
h2 
[1] FALSE FALSE FALSE  TRUE  TRUE
h1 & h2
[1] FALSE FALSE FALSE  TRUE  TRUE
h1 | h2
[1] FALSE  TRUE FALSE  TRUE  TRUE
!h1
[1]  TRUE FALSE  TRUE FALSE FALSE

For missing values, Boolean operations follow specific rules due to the ambiguity of NA, which could be either TRUE or FALSE. The result and reasoning are summarized in the tables below.

OR Logic (|)

X x | NA reasoning
TRUE TRUE At least one is TRUE - result is guaranteed TRUE
FALSE NA NA could be TRUE or FALSE - result is unknown
NA NA Both values unknown - result is unknown

AND Logic (&)

X x & NA reasoning
TRUE NA NA could be TRUE or FALSE - result is unknown
FALSE FALSE One is FALSE - result is guaranteed FALSE
NA NA Both values unknown - result is unknown

The %in% operator checks if elements of one vector are present in another, returning TRUE for matches. Unlike ==, NA %in% NA is TRUE.

c(1, 2, NA) %in% NA
[1] FALSE FALSE  TRUE
c(1, 2, NA) == NA
[1] NA NA NA
c(1:5, NA) %in% c(1, 5, 11)
[1]  TRUE FALSE FALSE FALSE  TRUE FALSE

2.6.4 The all() and any() Functions

The all() and any() functions summarize logical vectors. all() returns TRUE if all elements are TRUE, while any() returns TRUE if at least one element is TRUE. Missing values may affect results unless handled with na.rm=TRUE.

f <- c(3.01, 3.001, 3.0001, 3.00001)
any (f == 3.001, na.rm=TRUE)
[1] TRUE
any (f >3.1, na.rm=TRUE)
[1] FALSE
all (f > 3.001, na.rm=TRUE)
[1] FALSE

Instead of using the any() or all() functions, we can solve problems based on what we’ve learned. For example, the following commands illustrates how to test if any elements in F equal 3.001?

Instead of relying solely on any() or all(), you can use techniques learned earlier. For example, to test if any elements in f equal 3.001, follow these steps:

## Step 1: Compare f with 3.001 using ==
f == 3.001
[1] FALSE  TRUE FALSE FALSE
## Step 2: Count the number of elements in f equal to 3.001
sum(f == 3.001, na.rm=TRUE)
[1] 1
## Step 3: Check if the count is greater than 0
sum(f == 3.001, na.rm=TRUE) > 0
[1] TRUE

2.6.5 Numeric Summaries

Logical vectors are coerced to numeric values in arithmetic operations, with TRUE as 1 and FALSE as 0. This enables counting or proportion calculations, such as determining how many elements meet a condition. Additionally, mean() gives the proportion of TRUE values where x is a logical vector.

v <- c(seq(1, 6, by=0.5), 10)
mean_v <- mean(v, na.rm=TRUE)
mean_v
[1] 4.041667
v > mean_v
 [1] FALSE FALSE FALSE FALSE FALSE FALSE FALSE  TRUE  TRUE  TRUE  TRUE  TRUE
sum(v > mean_v)
[1] 5
mean(v > mean_v)
[1] 0.4166667

In this example, sum(v > mean_v) counts elements greater than the mean, and mean(v > mean_v) gives the proportion.

For a vector with missing values, use na.rm=TRUE to compute the proportion accurately.

v_with_na <- c(1, 2, 3, NA, 5)
mean(v_with_na > 3, na.rm=TRUE)
[1] 0.25

In this case, v_with_na > 3 yields [FALSE FALSE FALSE NA TRUE], and mean(v_with_na > 3, na.rm=TRUE) excludes the NA, giving a proportion of 0.25 (or 25%) based on the one TRUE value out of four non-missing elements.

2.7 Factors

Factors are a specialized type of vector in R, designed to represent categorical data with a fixed set of levels. Unlike character vectors, factors impose an order on their values, which is useful for statistical modeling and data analysis. This section covers the creation of factors, their sorting behavior, handling of invalid levels, and basic subsetting, building on the vector concepts from earlier sections.

2.7.1 Creating and Sorting Factors

Factors are created using the factor() function, which converts a vector (typically character) into a factor by defining its levels with the levels= argument. Without specified levels, factor() assigns levels alphabetically based on the input, which may vary across systems.

x1 <- c("Dec", "Apr", "Jan", "Mar")
factor(x1)
[1] Dec Apr Jan Mar
Levels: Apr Dec Jan Mar

To impose a meaningful order, provide a levels= argument. The sort() function then respects this order rather than alphabetical sorting. Controlling the levels is important for statistical models because the first level is treated as the baseline, influencing coefficient interpretation.

month_levels <- c(
  "Jan", "Feb", "Mar", "Apr", "May", "Jun",
  "Jul", "Aug", "Sep", "Oct", "Nov", "Dec"
)
y1 <- factor(x1, levels=month_levels)
sort(y1)
[1] Jan Mar Apr Dec
Levels: Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec
levels(y1)
 [1] "Jan" "Feb" "Mar" "Apr" "May" "Jun" "Jul" "Aug" "Sep" "Oct" "Nov" "Dec"
typeof(y1)
[1] "integer"
class(y1)
[1] "factor"
nlevels(y1)
[1] 12

To convert a factor to a character vector, you can use the as.character() function.

as.character(y1)
[1] "Dec" "Apr" "Jan" "Mar"

If the input contains invalid levels (not in levels=), R assigns NA to those elements.

x2 <- c("Dec", "Apr", "Jam", "Mar")
y2 <- factor(x2, levels=month_levels)
y2
[1] Dec  Apr  <NA> Mar 
Levels: Jan Feb Mar Apr May Jun Jul Aug Sep Oct Nov Dec

Note that sorting is not meaningful for factors without defined levels, and system-dependent alphabetical order can lead to inconsistent results.

2.7.2 Subsetting a Factor

You can extract elements from a factor using an index vector within square brackets. In the following example, a new factor is created containing only the first three elements of f. In f1, the oc level is empty but still part of the levels attribute, keeping the number of levels at 3. This can complicate interpretation in statistical functions.

f <- factor(c("la", "sb", "la", "oc"), levels=c("la", "sb", "oc"))
f1 <- f[1:3]
f1
[1] la sb la
Levels: la sb oc
nlevels(f1)
[1] 3

To remove unused levels, use the following methods:

factor(f[1:3])
[1] la sb la
Levels: la sb
f[1:3, drop=TRUE]
[1] la sb la
Levels: la sb

The drop=TRUE argument or reapplying factor() ensures unused levels are excluded, improving clarity in downstream analysis.

2.8 Matrices and Arrays

Matrices and arrays provide structured storage for data in tabular or multi-dimensional formats, extending the capabilities of vectors. This organization facilitates understanding and manipulation, especially in statistical and numerical computations. R offers optimized functions and operations for matrices and arrays, often outperforming vector-based approaches for larger datasets. This section explores their creation, concatenation, attributes, and basic element access, building on the vector foundation from earlier sections.

2.8.1 Creating a Matrix

A matrix is a two-dimensional array, created by assigning a dim attribute with two values (rows, columns) to a vector. The storage mode of the matrix matches that of its elements (e.g., double, integer).

v <- c(2.9, 3.5, 4.5, NA, 3, 2.4)
v
[1] 2.9 3.5 4.5  NA 3.0 2.4
dim(v) <- c(2,3)
v
     [,1] [,2] [,3]
[1,]  2.9  4.5  3.0
[2,]  3.5   NA  2.4

Alternatively, use the matrix() function, which offers more control. By default, it fills the matrix column-wise.

m <- matrix(v, nrow=2, ncol=3)
m
     [,1] [,2] [,3]
[1,]  2.9  4.5  3.0
[2,]  3.5   NA  2.4

To fill row-wise, set byrow=TRUE.

m1 <- matrix(v, 2, 3, byrow = TRUE)
m1
     [,1] [,2] [,3]
[1,]  2.9  3.5  4.5
[2,]   NA  3.0  2.4

The t() function transposes a matrix, swapping rows and columns.

t(m)
     [,1] [,2]
[1,]  2.9  3.5
[2,]  4.5   NA
[3,]  3.0  2.4

2.8.2 Concatenating Matrices

The cbind() and rbind() functions combine vectors or matrices by columns or rows, respectively. For cbind(), all arguments must have the same number of rows; for rbind(), the same number of columns is required.

n1 <- matrix(1:4, nrow=2)
n1
     [,1] [,2]
[1,]    1    3
[2,]    2    4
n2 <- matrix(5:10, nrow=2)
n2
     [,1] [,2] [,3]
[1,]    5    7    9
[2,]    6    8   10
cbind(n1, n2)
     [,1] [,2] [,3] [,4] [,5]
[1,]    1    3    5    7    9
[2,]    2    4    6    8   10
rbind(v, n2)
     [,1] [,2] [,3]
[1,]  2.9  4.5  3.0
[2,]  3.5   NA  2.4
[3,]  5.0  7.0  9.0
[4,]  6.0  8.0 10.0

If the number of rows or columns differs, cbind() and rbind() may recycle elements or throw an error. For mismatched dimensions, Base R does not provide direct functions to align them automatically. Instead, consider package functions: plyr::rbind.fill() from the plyr package and dplyr::bind_rows() from the dplyr package for row binding, which can handle varying column numbers by filling with NA where needed.

Note: Ensure compatible dimensions to avoid errors or unexpected recycling; use external packages for flexible concatenation.

2.8.3 Matrix Attributes

Matrices do not have a names attribute by default, but row and column names can be assigned using rownames() and colnames().

rownames(v) <- c("r1", "r2")
colnames(v) <- c("c1", "c2", "c3")
v
    c1  c2  c3
r1 2.9 4.5 3.0
r2 3.5  NA 2.4

The dimnames() function assigns both row and column names simultaneously.

dimnames(v) <- list(c("r1", "r2"), c("c1", "c2", "c3"))
v
    c1  c2  c3
r1 2.9 4.5 3.0
r2 3.5  NA 2.4

Use length() for the total number of elements, but nrow(), ncol(), and dim() provide more meaningful dimension details. The attributes() function lists all attributes.

length(v)
[1] 6
nrow(v)
[1] 2
ncol(v)
[1] 3
dim(v)
[1] 2 3
attributes(v)
$dim
[1] 2 3

$dimnames
$dimnames[[1]]
[1] "r1" "r2"

$dimnames[[2]]
[1] "c1" "c2" "c3"

2.8.4 Accessing Elements in a Matrix

Matrix elements are accessed using two index vectors within square brackets, denoted as a[m, n], where m selects rows and n selects columns. Separate m and n with a comma. Omitting m (e.g., a[, n]) selects all rows for the specified columns, and omitting n (e.g., a[m, ]) selects all columns for the specified rows.

v[2,3]
[1] 2.4
v[2,]
 c1  c2  c3 
3.5  NA 2.4 
v[,3]
 r1  r2 
3.0 2.4

Note: This basic access is foundational; advanced indexing is covered in Chapter 4.

2.8.5 Creating Arrays

Arrays are multi-dimensional extensions of matrices, created by specifying a dim attribute with a length of three or more. For example:

x <- 1:12
dim(x) <- c(2,2,3)
dimnames(x) <- list(c("r1", "r2"), 
                    c("cl", "c2"), 
                    c("pl", "p2", "p3"))
x
, , pl

   cl c2
r1  1  3
r2  2  4

, , p2

   cl c2
r1  5  7
r2  6  8

, , p3

   cl c2
r1  9 11
r2 10 12
attributes(x)
$dim
[1] 2 2 3

$dimnames
$dimnames[[1]]
[1] "r1" "r2"

$dimnames[[2]]
[1] "cl" "c2"

$dimnames[[3]]
[1] "pl" "p2" "p3"

Alternatively, use the array() function.

x <- array(1:12, c(2,2,3))
